# 安全策略

## 报告安全问题

请不要在公开 Issue 中提交以下内容：

- API Key、Token、Cookie
- 可直接访问的付费接口凭证
- 用户素材、转写原文或个人备注
- 能被直接利用的漏洞细节

请优先使用 GitHub 仓库的 Private Vulnerability Reporting 功能私密报告。

报告中建议包含：

1. 受影响版本或提交
2. 复现步骤
3. 可能造成的影响
4. 建议修复方式

## 密钥泄露处理

如果密钥曾经出现在代码、提交历史、Issue、截图或聊天记录中：

1. 立即到对应服务商后台撤销旧密钥
2. 生成新密钥
3. 更新本机 `backend/.env`
4. 检查 Git 历史，而不只是删除当前文件

`.gitignore` 不能保护已经提交过的密钥。

## 支持范围

优先处理当前主分支能够复现的安全问题。第三方服务自身的账户、计费和平台漏洞应同时报告给对应服务商。
